## 方案摘要

### 模块布局

```
mini_coding_agent/harness/
  nodes/
    locate.py     # files_hint + read_file + search（rg 回退）
    generate.py   # 1× complete → parse → run_tool(write/patch)
    verify.py     # test_command → run_shell；否则 py_compile
    review.py     # 1× complete → <final>
  executor.py     # 拓扑执行 + verify→generate retry（max 2）
  pipeline.py     # plan() + execute_dag()
  runner.py       # fix_bug + high → run_pipeline；失败/其他 intent → open
  types.py        # NodeResult / HarnessContext / PipelineResult / PIPELINE_INTENTS_V1
```

### 执行流程（fix_bug）

```
handle_ask(harness on)
  → classify_gate
  → intent=fix_bug & route=harness_pipeline
  → run_pipeline(agent, fix_bug, message)
       → plan() 填充槽位
       → execute_dag：locate → generate → verify → review
            verify fail → 回退 generate（≤2 次）
  → 成功：返回 review.final（不调用 agent.ask）
  → 失败：stderr 原因 + agent.ask() 降级
```

### 关键设计

| 项 | 实现 |
|----|------|
| generate 写盘 | `agent.run_tool("patch_file"\|"write_file")` → governance |
| verify | 有 `test_command` 则 `run_shell`；否则对改动 `.py` 做 `py_compile` |
| retry | executor 读模板 `retry.verify`；失败时 `index` 回退至 `generate` |
| stderr | `[harness] fix_bug 2/4 verify ok` |
| 其他四类 high | stderr「尚未实现」→ `agent.ask()` |
| RIG | 未接入（locate 仅用 slots + search） |
