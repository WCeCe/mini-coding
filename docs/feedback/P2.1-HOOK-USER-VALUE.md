## 方案摘要

### 三层 Hook 栈挂接

```
MiniAgent.__init__
  → load_hook_config(<workspace>/.mini-coding-agent/hooks.yaml)   # 第 3 层
  → apply_cli_overrides(args)                                     # CLI 覆盖 YAML
  → register_builtin_hooks(hook_registry, hook_config)            # 第 2 层装配
       ├─ session_trace  → ToolTraceHook        (Phase 2 保留)
       ├─ trace_display  → TraceDisplayHook    (第 1 层终端可见)
       └─ shell_audit    → ShellAuditHook      (run_shell 模式告警)
  → run_tool → _invoke_tool_with_hooks → emit_pre / emit_post
```

- **第 1 层（运行时可见）**：`TraceDisplayHook.post_tool` 向 **stderr** 打印一行摘要，与模型 `final` 的 **stdout** 区分。
- **第 2 层（内置 Hook 价值）**：`ToolTraceHook` 写 `session["tool_trace"]`；`ShellAuditHook` 对 `run_shell` 做正则匹配，stderr 告警 + 写 `session["shell_audit"]`；均 **不阻断**、不跳过 `approve()`。
- **第 3 层（YAML 装配）**：`hook_config.py` 解析 `.mini-coding-agent/hooks.yaml`，`builtin_hooks.py` 按开关注册；`enable_trace_hook=False` 时关闭全部内置 Hook（测试隔离，与 Phase 2 一致）。

### 终端输出格式

| 类型 | 通道 | 格式示例 |
|------|------|----------|
| Trace 展示 | stderr | `[mini-agent] #1 read_file ok 3.2ms` |
| Shell 审计 | stderr | `[mini-agent] SHELL AUDIT: matched 'rm -rf' — cmd: rm -rf /tmp/demo` |
| 配置告警 | stderr | `[mini-agent] hooks config: hooks.yaml unreadable ...; using defaults` |

- 成败：`ok` / `FAIL`；耗时保留 1 位小数 ms。
- Shell 命令展示用 `middle(..., 80)` 截断，避免泄露过长敏感参数。

### YAML schema

默认路径：`<workspace_root>/.mini-coding-agent/hooks.yaml`

```yaml
builtin_hooks:
  session_trace: true    # ToolTraceHook → session["tool_trace"]
  trace_display: true    # TraceDisplayHook → stderr 每步一行
  shell_audit: true      # ShellAuditHook → run_shell 模式审计

shell_audit:
  warn_patterns:         # 正则列表（re.IGNORECASE）
    - "rm -rf"
    - "curl.*\\|.*sh"
```

- 缺省文件 → 三项全 `true`，`warn_patterns` 用内置默认。
- 根节点须为 mapping；未知键忽略；布尔字段类型错误 → 该字段回退默认 + stderr 告警。
- 无效正则在 `ShellAuditHook` 初始化时跳过（fail-open）。

### CLI 优先级

**CLI 显式旗标 > YAML > 内置默认**

| CLI 旗标 | 效果 |
|----------|------|
| `--no-trace-display` | 关闭终端 trace 行 |
| `--no-session-trace` | 关闭 session `tool_trace` |
| `--no-shell-audit` | 关闭 shell 审计 Hook |
| `--hooks-config PATH` | 指定 hooks.yaml 路径（否则用 workspace 默认路径） |

- 未传旗标时不覆盖 YAML 对应项。
- `cli.build_agent` 在构造 `MiniAgent` 前完成 load + override；`MiniAgent(..., hook_config=...)` 直接使用合并结果。
- `delegate` 子 Agent 继承父 `hook_config`，保证栈行为一致。

### 新增模块

| 模块 | 职责 |

| `hook_config.py` | `HookConfig`、YAML 加载、CLI 覆盖、fail-open 告警 |
