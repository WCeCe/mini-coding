### 模块布局

```
mini_coding_agent/harness/
  types.py              # 扩展 DagNode / DagSlots / DagInstance / RetryPolicy / NODE_TYPES
  slots.py              # 规则槽位填充（goal、files_hint、symbols_hint、test_command、ops_allowlist）

  templates/
    generate_code.json  # locate → generate → verify → review + retry
    fix_bug.json        # 同上
    refactor.json       # locate → plan → generate → verify → review + retry
    explain.json        # locate → explain（只读）
    project_ops.json    # locate → ops → review
```

### Planner 流程

```
plan(intent_id, user_message, skill_name?, workspace_root?)
  → 校验 intent_id ∈ INTENT_IDS（否则 ValueError）
  → load_template(intent_id)  # JSON 静态拓扑 + retry
  → fill_slots(...)           # 规则提取，无 LLM
  → DagInstance(intent_id, nodes, slots, retry)
```

### 槽位规则（首版）

| 槽位 | 来源 |
|------|------|
| `goal` | 用户消息截断（300 字符，`GOAL_MAX_LEN`） |
| `files_hint` | traceback `File "…"` + 消息内 `*.py` 等路径正则 |
| `symbols_hint` | `NameError` / `AttributeError` / `ImportError` 等 |
| `test_command` | 工作区存在 `tests/`、`pytest.ini` 或 pyproject 含 pytest → `python -m pytest -q` |
| `skill_name` | Gate JSON（Planner 参数传入） |
| `ops_allowlist` | `project_ops` 专用，`DEFAULT_OPS_ALLOWLIST` |

### 模板注册

- 五类 intent 与 `templates/<intent_id>.json` **一一映射**
- 加载时校验：`intent_id` 与文件名一致、节点 `type` ∈ `NODE_TYPES`、deps 为数组

### 明确未做（留 5.3+）

- `executor`、`pipeline`、节点 run、`rig/`、runner 接 pipeline
