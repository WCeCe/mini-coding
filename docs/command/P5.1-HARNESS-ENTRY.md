## 目标

交付 Graph Harness **编排入口**与 **LLM 意图分类（Gate）**：每次用户 ask 先分类为封闭五类之一；`confidence=low` 或非法 intent 走 **open**（`agent.ask()`）。本阶段 **不实现** DAG 执行（`harness_pipeline` 可打日志后 fallback open）。

达到 [`struct/phase5-graph.md`](../struct/phase5-graph.md) 子阶段 5.1 与 **M1**。

---

## 约束

- 见 [`PHASE5-OVERVIEW.md`](./PHASE5-OVERVIEW.md) §3
- **意图封闭 5 类**：`generate_code`、`fix_bug`、`refactor`、`explain`、`project_ops`（定义见 struct §5.1）
- **Gate = 每 ask 1 次** `model_client.complete`；专用 prompt；输出 JSON：`intent_id`、`confidence`（`high`|`low`）、可选 `skill`
- **测试禁止依赖 Ollama**；Gate 用 `FakeModelClient`
- **规则兜底仅**：JSON 解析失败、intent 不在五类 → `low`（不得用规则替代 LLM 做主分类）
- CLI：`--gate-log`（打印 GateResult）；`--harness off|on`（5.1 时 `on` 仍可无流水线，须安全 fallback open）
- session：写入 `last_gate`（字段名可在 feedback 说明）
- **不实现**：`templates/`、`executor`、`rig/`、节点
